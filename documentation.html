Documentation

Info about the api calls, the responses and error codes

/api/items

GET

Description:

Sends all items in stock.

Response:

200 - ok. The response would be:
[
{
id: 0,
name: 'Candyfloss',
description: 'Sweet and tasty sugar',
price: 12,
currency: 'EUR',
quantity: 350
}
]

POST

Description:

Create a new item. The body should contain item details: name, description, price, currency and quantity.

Response:

200 - ok. The response would be the id of the new item.

400 - bad request. If the body of the request is empty, or if some fields are not defined

/api/items/archive

PUT

Description:

Archive an item. It is still saved in the database, but not sent in GET request. Query should have item id. Fx '/api/items/archive?item=1'.

Response:

200 - ok. The item has been archived.

400 - bad request. No item specified

404 - Item was not found

/api/items/quantity

PUT

Description:

Set a new quantity for an item. Query should contain item, and new quantity, which must be positive. Fx '/api/items/quantity?item=1&quantity=20'.

Response:

200 - ok. The item has been updated.

400 - bad request. No item specified, no quantity specified, or negative quantity

404 - Item was not found

/api/users

GET

Description:

Gets the id of a user. The query should specify an email. Fx '/api/users&email=example@example.com'

Response:

200 - ok. The response is the id of the user.

400 - bad request. No email specified

404 - not found. User does not exist

POST

Description:

Creates a new user. The query should specify an email and a name. Fx '/api/users&email=example@example.com&name=john'

Response:

200 - ok. The user was created The response is the id of the new user.

400 - bad request. No email or no name specified, or invalid email

409 - duplicate entry. User already exists

/api/orders

GET

Description:

Get all orders placed by a user. Query needs user as input, fx '/api/orders?user=1'

Response:

200 - ok. The response could look like:

[
{
items: {[
amount: 1,
name: 'Stroopwaffel'
]},
order_time: 'Thu Jul 23',
orderid: 1,
total_price: 1
}
]

400 - bad request. No user specified.

POST

Description:

Place order. The input should be a userId, together with a list of itemId's in the body.'

Response:

200 - ok. The response is the new orderId

400 - bad request. No user or no items specified. The list of items cannot be empty
